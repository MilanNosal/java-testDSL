NetBeans Environment

Section Creating new test we saw how to work in NetBeans to create a new test. In this section we will take a look at some awesome features that can be used during creating the test.

Code completion

One of the most powerful features of the editor is code completion. In the Test-it! language you can use this feature to define the test faster and more reliably. When a typing cursor is on a place, where a command should be used, you can press Ctrl + Space to have the editor present a menu with options that can be typed in that place. For example, see following figure where I started to type in the begining of the command for question (in Slovak question is 'otazka').

Moreover if you browse the options using up and down arrows the editor shows also the documentation to the given command (blue rectangle).

Selecting an option can be confirmed pressing an Enter keyboard button or clicking a selected command. After that the editor generates a usage skeleton of the given command as you can see in following figure (I selected single choice question - 'otazka_s_jednou_spravnou_odpovedou' in Slovak):

In the generated parentheses you can see placeholders for parameters - question text that has to be typed in, and the points for the question defaulted to 0.

Be careful about parentheses! The language is sensitive to correct usage of parentheses - if you open a parenthesis (you use a left parenthesis '('), you have to close it as well (use it pair right parenthesis ')'). The editor generates parentheses pairs nicely, but be careful not to delete one, or misplace them, because that can mess up the test definition.

Error support

If you do not keep all the rules for test definition, an error wil occur and the test will not be runnable. In such a case an error is reported by the editor. For example, we might by accident define two correct answer options in a single choice question. In this case if we would try to run the test we would get following error saying that there cannot be another correct answer option ('spravna_odpoved' in Slovak) in a single choice question ('otazka_s_jednou_spravnou_odpovedou' in Slovak):

As you can see, the environment provides a clickable navigation link that puts cursor on the line with the error. The error description thus specifies the line in which the error was detected, and what is the error about. It also suggests possible solution (in this case removing the second correct option, or changing the question type to a one that supports mutliple correct answers). A clickable link helps you to identify the erroneous line faster.
