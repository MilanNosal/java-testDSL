Creating your first test

Running the editor

You will be working with the NetBeans environment. If it is not already open, run it by doubleclick on its icon on the desktop. You should see the window that is shown on the following figure. Take note of the view on the top left with the "Projects" title, then the "Output" view on the right bottom, and finally the largest view on the right top - the editor. In the "Projects" view you can create new files with the tests, in the "Output" window you can see if there are any error messages (only if you do a mistake), and in the editor view you will see and edit the test.

New test

You can create a new test in a following way:

  1. In the Projects view expand the TestDSLJava folder, in it expand Source Packages folder and right-click on the test folder
  2. in the menu that pops out select New/Other
  3. In the wizard window select the DSL category and its item Test-it! and click Next
  4. In the next window fill the File Name with the name of the file to be created (without spaces, e.g. 'NewTest'), and click Finish.

After these steps the main editor view opens a new file with a skeleton of a sample test (notice that in the Projects view test folder a new file was created).

The created file can be edited to change the sample test to a wanted one, or you can delete the sample and write the test from the scratch. In next section I will discuss more about what you can use to define the test.
